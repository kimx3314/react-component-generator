## 개요

이 스킬은 `git diff`를 분석하여 변경사항을 논리적 단위로 분류하고, Conventional Commits 형식의 한국어 메시지로 자동 커밋합니다.

## 절차

### 1단계: 변경사항 분석
```bash
git diff --cached     # 스테이징된 변경사항
git diff              # 스테이징 안 된 변경사항
git status            # 전체 상태 확인
```

- 변경된 파일 목록 파악
- 각 파일의 변경 내용 분석
- 논리적 단위 분류 (기능 추가/버그 수정/리팩토링/설정 변경)

### 2단계: 논리적 단위 분류

변경사항들을 다음 카테고리 중 하나로 분류:

| 카테고리 | 설명 | 예시 |
|---------|------|------|
| **feat** | 새로운 기능 추가 | 새 컴포넌트, API 엔드포인트 |
| **fix** | 버그 수정 | 오류 개선, 예외 처리 |
| **refactor** | 기능 변경 없이 코드 개선 | 변수명 변경, 로직 정리 |
| **chore** | 설정, 의존성, 빌드 등 | package.json, 설정 파일 |
| **docs** | 문서 변경 | README, 주석 |
| **style** | 코드 스타일 (포맷팅, 세미콜론 등) | Prettier, ESLint |
| **test** | 테스트 추가/수정 | 단위 테스트, E2E 테스트 |

**분류 규칙:**
- 여러 카테고리가 섞여 있으면 주 변경사항 기준으로 분류
- 논리적으로 독립적인 변경이면 별도 커밋으로 분리 (최대 3-4개)
- 모두 같은 카테고리면 하나의 커밋으로 통합

### 3단계: 한국어 커밋 메시지 작성

**형식:**
```
<type>(<scope>): <subject>

<body>
```

**규칙:**
- `<type>`: feat, fix, refactor, chore, docs, style, test 중 하나
- `<scope>`: 선택사항. 변경 범위 (예: auth, api, ui) - 여러 파일이면 생략
- `<subject>`: 현재형, 명령조 사용. 마침표 없음. 50자 이내
- `<body>`: 선택사항. 자세한 설명 필요한 경우만 (빈 줄로 분리)

**예시:**
```
feat(auth): JWT 기반 인증 구현

- 사용자 로그인/로그아웃 기능 추가
- 토큰 갱신 로직 구현
- 인증 미들웨어 적용

feat: Google 모델 실패 시 자동 폴백 추가

fix(ui): 반응형 레이아웃 버그 수정

refactor: 변수명 정규화 (_id → id)

chore: 의존성 업데이트
```

### 4단계: 사용자 승인 및 커밋 실행

1. **커밋 계획 제시**
   - 예정된 커밋 목록 (type, scope, message)을 테이블로 표시
   - 각 커밋에 포함될 파일 명시

2. **사용자 승인 요청**
   - "이 방식으로 진행할까요?" 라고 확인
   - 수정 요청 대응 (메시지 변경, 분류 재구성 등)

3. **커밋 실행**
   ```bash
   git commit -m "<message>"
   ```
   - 각 커밋을 순서대로 실행
   - 실패 시 원인 분석 및 사용자 안내

## 주의사항

- **스테이징 상태 확인**: 스테이징되지 않은 파일은 `git add` 후 커밋
- **브랜치 확인**: 현재 브랜치가 올바른지 확인
- **푸시 전 검토**: 커밋 완료 후 푸시 권유 (자동 푸시는 하지 않음)
- **중요한 변경**: `git diff`를 시각적으로 검토하고 의도치 않은 변경 제외

## 실행 예시

**입력:**
```
사용자: "변경사항 커밋해줘"
```

**처리 과정:**
```
1. git diff 분석
   - src/auth/login.ts (기능 추가)
   - src/components/Button.tsx (버그 수정)
   - package.json (의존성 업데이트)

2. 커밋 계획 수립
   | # | Type | Scope | Message | 파일 |
   |---|------|-------|---------|------|
   | 1 | feat | auth | JWT 인증 로직 추가 | src/auth/login.ts |
   | 2 | fix | ui | 버튼 스타일 버그 수정 | src/components/Button.tsx |
   | 3 | chore | - | 의존성 업데이트 | package.json |

3. 사용자 승인 → 커밋 실행
```

## 실패 케이스 처리

- **스테이징된 파일 없음**: "커밋할 변경사항이 없습니다. 먼저 파일을 수정하거나 `git add`로 스테이징하세요."
- **커밋 실패**: Git 에러 메시지 분석 및 해결책 제시
- **분류 모호**: 사용자에게 의도 확인 후 재분류
